# Custom Design Systems

Place your custom DESIGN.md files here in subdirectories named after the style:

```
designs/
├── my-custom-dark/
│   └── DESIGN.md
├── corporate-blue/
│   └── DESIGN.md
└── ...
```

Each DESIGN.md should follow the [Stitch format](https://stitch.withgoogle.com/docs/design-md/format/).

## Minimum Required Sections

1. **Color Palette & Roles** — hex values with semantic names
2. **Typography Rules** — font families and hierarchy
3. **Component Stylings** — buttons, cards, inputs
