## 任务目标

实现统计页或首页卡片，展示语音协作次数、累计口述时间、口述生成字数、预计节省时间、常用人格。

## 实际改动

本任务在执行时发现统计卡片已在 T009 任务中完整实现。当前 `src/main.tsx` 的 870-930 行包含完整的统计卡片，包括：

1. **语音协作次数**（889-896 行）
   - 显示成功生成结果的历史记录总数
   - 使用 `BarChart3Icon` 图标
   - 数据来源：`historyStats.total_count`

2. **累计口述时间**（897-903 行）
   - 显示所有历史记录的录音时长总和
   - 使用 `Clock3Icon` 图标
   - 格式化为"X 分 Y 秒"
   - 数据来源：`historyStats.total_duration_ms`

3. **口述生成字数**（904-910 行）
   - 显示所有整理后文本的字数总和
   - 使用 `PencilIcon` 图标
   - 数据来源：`historyStats.total_output_chars`

4. **预计节省时间**（911-917 行）
   - 按每分钟手动输入 80 个中文字估算
   - 使用 `HistoryIcon` 图标
   - 格式化为"X 分 Y 秒"
   - 数据来源：`historyStats.estimated_saved_ms`

5. **常用人格**（918-930 行）
   - 显示使用次数最多的人格名称
   - 显示该人格的使用次数
   - 使用 `Mic2Icon` 图标
   - 数据来源：`historyStats.top_persona_name` 和 `historyStats.top_persona_count`

统计卡片采用响应式网格布局：
- 小屏幕：2 列
- 大屏幕：5 列
- 常用人格卡片在小屏幕占 2 列，大屏幕占 1 列

后端统计命令已在 T009 实现：
- `history_statistics`：返回 `HistoryStatistics` 结构体

后端统计计算逻辑（`src-tauri/src/data.rs`）：
- `total_count`：历史记录总数
- `total_duration_ms`：录音时长总和
- `total_output_chars`：生成字数总和
- `estimated_saved_ms`：按每分钟 80 字估算的节省时间
- `top_persona_name` 和 `top_persona_count`：使用次数最多的人格

因此本任务无需新增代码，仅需验证现有功能完整性。

## 为什么这么做

T009 任务在实现历史记录功能时，同步实现了统计卡片。这符合项目的开发方式：历史记录和统计数据紧密相关，统计数据由历史记录实时计算，因此在同一个任务中实现可以确保数据一致性。

统计卡片作为"语音输入成效"卡片展示在主界面中，而不是独立页面，原因：
1. 统计数据是用户使用工具的效果反馈，应该在主界面中直观展示
2. 统计卡片和历史记录列表在同一个卡片中，方便用户同时查看数据和详情
3. 减少页面切换，提升用户体验
4. 符合当前单页应用的设计模式
