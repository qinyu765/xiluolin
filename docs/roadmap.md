# XiLuoLin 项目路线图

> 路线图表达当前方向和优先级，不承诺固定版本或交付日期。具体工作以 Issue、Pull Request 和发布计划为准。

## 长期目标

把 XiLuoLin 建设为一个可靠、可配置、尊重本地数据所有权的开源桌面语音输入助手，让用户能够在真实工作流中将自然口述快速转换为可直接使用的文本。

## 当前优先级

### P0：可靠的核心输入链路

- 完善录音状态、取消、超时和异常恢复。
- 验证全局快捷键在 Windows 与 macOS 上的冲突和权限行为。
- 验证键盘注入、自动粘贴和剪贴板降级路径。
- 增加 ASR、文本整理、历史保存和统计更新的自动化测试。
- 明确临时录音生命周期和失败清理策略。

### P1：可用性与配置体验

- 恢复或重新设计首页语音输入入口。
- 改善首次启动、Provider 配置和权限引导。
- 提供清晰的处理中、成功、失败和可重试状态。
- 完善人格、热词和历史的搜索、导入导出与管理体验。
- 增加可访问性、键盘操作和多语言界面基础。

### P1：Provider 与本地能力

- 稳定 Provider 接口和错误模型。
- 支持更多 ASR 与 OpenAI-compatible 文本服务。
- 评估本地离线 ASR 和本地文本模型的资源、隐私与体验取舍。
- 为 Provider 增加连接测试、超时、重试和速率限制提示。

### P2：发布与开源协作

- 建立版本号、变更日志和安装包发布流程。
- 补充 Windows、macOS 的兼容性矩阵和安装说明。
- 建立 Issue 模板、PR 模板和贡献者友好的开发任务。
- 持续整理架构决策、数据迁移和安全审查记录。

## 当前非核心范围

以下方向不是当前核心优先级，但欢迎通过 Issue 提供明确使用场景：

- 完整系统输入法内核
- 会议纪要和长音频批量转写
- 多人实时协作平台
- 在本仓库内训练通用语音模型

这些边界用于保持维护焦点，并不代表永久拒绝。提案会根据用户价值、隐私、复杂度、维护成本和架构一致性评估。
