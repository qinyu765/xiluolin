# 历史开发记录

本目录保存 XiLuoLin 在不同开发阶段的任务拆分、实现记录和验证结论，用于追踪技术演进。

其中部分文档形成于项目早期，可能包含 MVP、demo、比赛、评审、`dev` 分支或直接在 `main` 开发等旧表述。这些内容只代表当时背景，不再定义当前项目方向、产品范围或协作流程。

当前文档入口：

- 项目定位与状态：[`README.md`](../../README.md)
- 开源贡献流程：[`CONTRIBUTING.md`](../../CONTRIBUTING.md)
- 产品方向：[`docs/roadmap.md`](../roadmap.md)
- 产品需求：[`docs/requirements-analysis.md`](../requirements-analysis.md)
- 技术方案：[`docs/solution-design.md`](../solution-design.md)
- 使用与验证：[`docs/usage-guide.md`](../usage-guide.md)

新增工作应通过 Issue 和 Pull Request 管理；需要长期保留的产品或架构决策，应整理到 `docs/` 的当前文档，而不是继续依赖早期任务记录作为唯一事实源。
