# T016 实现历史页

> **归档说明：** 本文记录特定开发阶段的背景与决策，其中的 MVP、demo、比赛或旧分支流程表述仅用于保留历史，不代表 XiLuoLin 当前的开源项目定位与协作方式。当前信息请以根目录 `README.md`、`CONTRIBUTING.md` 和 `docs/roadmap.md` 为准。

## 任务目标

为历史记录列表添加复制功能，让用户可以快速复制历史记录的整理结果。

## 实际改动

### 前端改动 (src/main.tsx)

1. **新增复制历史记录函数**：
   - `handleCopyHistoryText(text: string)`: 复制指定的历史记录文本到剪贴板
   - 复制成功后更新 `historyStatus` 状态提示

2. **改造历史记录卡片**：
   - 在每条历史记录右上角添加复制按钮
   - 使用 `ghost` 变体的小型按钮，保持界面简洁
   - 按钮显示复制图标（CopyIcon）
   - 调整布局，使用 `flex-1` 让文本区域占据剩余空间

## 为什么这么做

### 功能设计

1. **复制按钮位置**：
   - 放在每条历史记录的右上角
   - 与输出模式标签并列显示
   - 使用 ghost 变体，不抢占视觉焦点

2. **复用现有 UI**：
   - 历史记录功能已在 T009 中实现
   - 只需添加复制功能，无需重构整个页面
   - 保持与主界面复制按钮的一致性

3. **状态反馈**：
   - 复制成功后在 `historyStatus` 区域显示提示
   - 与历史记录加载状态使用同一个状态字段
   - 用户可以清楚知道操作结果

### 实现简化

1. **不创建独立页面**：
   - 当前单页应用结构简单
   - 历史记录已在主页面展示
   - MVP 阶段优先功能完整性，不做过度拆分

2. **复用剪贴板 API**：
   - 使用与主界面相同的 `navigator.clipboard.writeText`
   - 保持错误处理逻辑一致

## 执行复盘

### 顺利的部分

1. **功能复用**：历史记录展示已在 T009 实现，只需添加复制功能
2. **代码简洁**：新增函数和 UI 改动都很小，易于维护
3. **类型安全**：TypeScript 确保参数类型正确

### 设计决策

1. **不创建独立页面**：
   - 当前应用是单页结构，所有功能在一个页面展示
   - 历史记录已经有独立卡片，功能完整
   - 后续如需扩展（搜索、筛选），可以再考虑独立页面

2. **复制按钮样式**：
   - 使用 `ghost` 变体，视觉上更轻量
   - 尺寸为 `sm`，与卡片整体比例协调
   - 图标大小 `size-3.5`，比标准图标略小

### 改进建议

1. **展开查看原始文本**：
   - 当前只显示整理后的文本（line-clamp-3）
   - 可以添加展开按钮，查看完整文本和原始识别文本
   - 作为后续优化项

2. **批量操作**：
   - 可以添加全选和批量删除功能
   - 作为后续优化项

3. **搜索和筛选**：
   - 按人格筛选
   - 按日期范围筛选
   - 关键词搜索
   - 作为后续优化项
