# T017 实现人格页

## 任务目标

实现人格页,展示内置人格列表、自定义人格列表、新建人格、编辑自定义人格、设置默认人格。

## 实际改动

### 后端改动

1. **新增 `PersonaDraft` 结构体** (`src-tauri/src/data.rs`)
   - 定义创建和更新人格时的数据结构
   - 包含 `name`、`description`、`scene`、`tone`、`output_structure`、`prompt` 字段

2. **新增数据库方法**
   - `create_persona`: 创建自定义人格,自动生成 UUID,设置 `is_builtin = false` 和 `is_default = false`
   - `update_persona`: 更新自定义人格,只允许更新 `is_builtin = false` 的人格
   - `delete_persona`: 删除自定义人格,只允许删除 `is_builtin = false` 的人格
   - `get_persona`: 根据 ID 查询单个人格

3. **新增 Tauri 命令**
   - `create_persona`: 前端调用创建人格
   - `update_persona`: 前端调用更新人格
   - `delete_persona`: 前端调用删除人格并返回更新后的人格列表

4. **注册命令** (`src-tauri/src/lib.rs`)
   - 在 `invoke_handler` 中注册 `create_persona`、`update_persona`、`delete_persona`

### 前端改动

1. **新增类型定义** (`src/main.tsx`)
   - `PersonaDraft`: 人格草稿类型,用于新建和编辑表单

2. **新增状态管理**
   - `personaDraft`: 当前编辑的人格草稿
   - `editingPersonaId`: 当前编辑的人格 ID,`null` 表示新建
   - `isPersonaDialogOpen`: 人格对话框开关
   - `isPersonaSaving`: 人格保存中状态

3. **新增处理函数**
   - `openCreatePersonaDialog`: 打开新建人格对话框
   - `openEditPersonaDialog`: 打开编辑人格对话框
   - `handleSavePersona`: 保存人格(新建或更新)
   - `handleDeletePersona`: 删除人格
   - `handleSetDefaultPersona`: 设置默认人格

4. **新增人格页 UI**
   - 内置人格列表:展示所有 `is_builtin = true` 的人格,显示名称、描述、场景、语气、输出结构
   - 自定义人格列表:展示所有 `is_builtin = false` 的人格,提供编辑和删除按钮
   - 默认人格标记:当前默认人格显示"默认"标签
   - 设为默认按钮:非默认人格显示"设为默认"按钮
   - 新建人格按钮:右上角操作区

5. **新增人格编辑对话框**
   - 人格名称输入框
   - 人格描述文本域
   - 适用场景输入框
   - 输出语气输入框
   - 输出结构输入框
   - 整理提示词文本域
   - 保存和取消按钮

## 为什么这么做

### 数据安全设计

- **只允许操作自定义人格**:内置人格由系统维护,不允许用户编辑或删除,避免破坏系统预设配置
- **数据库层面校验**:`update_persona` 和 `delete_persona` 在 SQL 中使用 `WHERE is_builtin = 0` 确保只操作自定义人格
- **前端 UI 区分**:内置人格和自定义人格分区展示,内置人格不显示编辑和删除按钮

### 用户体验设计

- **表单验证**:保存前检查必填字段(名称、描述、提示词),避免无效数据
- **状态反馈**:保存中显示加载动画,操作完成后更新状态提示
- **对话框滚动**:人格编辑对话框内容较多,设置 `max-h-[90vh] overflow-y-auto` 确保小屏幕可用
- **默认人格切换**:点击"设为默认"后同步更新 `AppConfig` 和数据库,确保主界面人格选择器和人格页状态一致

### 数据流设计

- **创建流程**:前端 → `create_persona` → 数据库插入 → 返回新人格 → 前端重新加载人格列表
- **更新流程**:前端 → `update_persona` → 数据库更新 → 返回更新后人格 → 前端重新加载人格列表
- **删除流程**:前端 → `delete_persona` → 数据库删除 → 返回更新后人格列表 → 前端更新状态
- **设为默认流程**:前端 → `set_default_persona` → 数据库更新 `is_default` → 更新 `AppConfig` → 返回更新后人格列表 → 前端同步状态
