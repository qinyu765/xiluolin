# 安全策略

## 支持范围

XiLuoLin 当前处于持续开发阶段。安全修复优先应用于 `main` 和最新发布版本；旧版本可能不会获得单独补丁。

## 报告漏洞

请不要公开提交包含漏洞细节、API Key、私人录音、用户文本或可直接利用步骤的 Issue。

优先使用 GitHub 仓库的 Private vulnerability reporting / Security Advisory 功能提交报告。若该功能不可用，请创建一个不包含漏洞细节的 Issue，说明需要与维护者私下联系。

报告建议包含：

- 受影响版本或 commit
- 操作系统与运行环境
- 漏洞影响和攻击前提
- 脱敏后的复现步骤或概念验证
- 建议的缓解方式（如有）

维护者会尽快确认报告、评估影响并协调修复与披露时间。

## 敏感数据边界

XiLuoLin 会处理麦克风音频、识别文本、模型 API Key 和本地历史。任何相关改动都应遵守以下原则：

- API Key 只能进入系统凭据库，不得写入日志或普通配置文件。
- 临时录音只能在受控目录内创建和清理。
- 日志、错误信息和遥测不得包含完整用户文本、完整录音路径或凭据片段。
- 新增网络请求必须在文档和 UI 中明确数据接收方与用途。
