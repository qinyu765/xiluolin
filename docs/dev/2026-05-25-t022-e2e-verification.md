## 目标

完成首次启动配置、短语音输入、人格切换、热词生效、历史保存、统计更新的完整闭环验证，确保 MVP 可演示、可解释、可复现。

## 背景

根据 `solution-design.md` 第 11 节，MVP 验收需要覆盖完整的用户路径，确保产品可以在评委面前流畅演示。

## 需求

### 验收路径

1. 首次启动后可以配置智谱 GLM-ASR-2512 和 OpenAI 文本模型
2. 用户可以选择默认人格
3. 用户可以创建自定义人格
4. 用户可以完成一次短语音输入
5. 系统可以返回 GLM-ASR-2512 的原始识别文本
6. 系统可以根据当前人格生成整理后的文本
7. 用户可以复制最终文本
8. 自动粘贴失败时复制结果仍可用
9. 历史记录成功保存
10. 统计卡片随历史记录更新
11. 热词可以作为上下文参与文本整理
12. API Key 不出现在 Git 跟踪文件中

### 性能和体验优化

1. 录音到结果展示的端到端延迟控制在合理范围
2. 界面响应流畅，无明显卡顿
3. 错误提示清晰、可操作
4. 快捷键响应及时

## 技术方案

### 验证方式

1. 按验收路径逐项手动测试
2. 记录测试结果和发现的问题
3. 修复问题后重新验证
4. 准备演示脚本和样例音频

### 调优方向

1. ASR 和文本整理的超时处理
2. 界面加载状态和进度反馈
3. 错误恢复和重试机制
4. 快捷键冲突检测和提示

## 实现步骤

1. 准备测试环境（配置真实 API Key、准备样例音频）
2. 按验收路径逐项测试
3. 记录问题和优化点
4. 修复问题
5. 重新验证
6. 准备演示脚本
7. 录制演示视频（可选）
