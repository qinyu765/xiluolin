# T023 实现导航框架

## 任务目标

建立左侧 Tab 导航栏和页面路由系统,为 UI 重构奠定基础。

## 实际改动

### 前端改动 (src/main.tsx)

1. **新增页面类型和状态**:
   ```typescript
   type Page = 'home' | 'persona' | 'hotword' | 'settings';
   const [currentPage, setCurrentPage] = useState<Page>('home');
   ```

2. **新增左侧导航栏组件**:
   - 使用 Tabs 组件实现垂直导航
   - Tab 项:首页、人格、热词、设置
   - 固定在左侧,宽度约 200px

3. **重构主布局**:
   ```
   ┌─────────────────────────────────────────┐
   │  XiLuoLin                      [设置]   │
   ├──────┬──────────────────────────────────┤
   │ 首页 │                                  │
   │ 人格 │        主内容区                  │
   │ 热词 │                                  │
   │ 设置 │                                  │
   └──────┴──────────────────────────────────┘
   ```

4. **页面内容条件渲染**:
   - 根据 currentPage 状态渲染对应页面内容
   - 暂时保留现有所有卡片,后续任务逐步重构

5. **移除冗余文本**:
   - 删除首页顶部的长描述:"面向办公、写作和编程场景..."
   - 保留 XiLuoLin 标题

## 为什么这么做

### 架构选择

1. **使用状态管理而非路由库**:
   - 单页应用,不需要 URL 路由
   - 避免引入额外依赖
   - 状态管理更简单直接

2. **左侧垂直 Tab**:
   - 符合桌面应用导航习惯
   - 为未来扩展预留空间
   - 视觉层级清晰

3. **渐进式重构**:
   - 先建立导航框架
   - 再逐页重构内容
   - 降低风险,便于验证

### UI 设计

1. **导航栏宽度**:
   - 200px 足够容纳中文标签
   - 不会过度占用主内容区空间

2. **保留顶部设置按钮**:
   - 快速访问全局设置
   - 与左侧导航的"设置"页不冲突
   - 设置按钮打开弹窗,设置页展示完整配置

## 执行复盘

### 顺利的部分

1. **类型系统**: TypeScript 和 Rust 的类型检查帮助提前发现问题
2. **组件复用**: shadcn/ui 的 Tabs 组件开箱即用,样式统一
3. **渐进式重构**: 采用方案A(先建立框架,后填充内容)降低了风险

### 遇到的问题

1. **文件规模**: main.tsx 超过 2000 行,一次性重构风险高
2. **JSX 结构**: 初次尝试移动卡片时破坏了 JSX 结构,导致编译错误
3. **策略调整**: 从完整重构调整为分步实施

### 改进建议

1. **组件拆分**: 后续应将 main.tsx 拆分为多个组件文件
2. **状态管理**: 考虑使用 Context 或状态管理库减少 prop drilling
3. **路由方案**: 当前使用状态管理实现路由,未来可考虑引入路由库
