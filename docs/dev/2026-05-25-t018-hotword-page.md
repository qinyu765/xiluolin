## 任务目标

实现热词页，展示热词列表、新增热词、编辑热词、启用或停用热词。

## 实际改动

本任务在执行时发现热词管理界面已在 T005 任务中完整实现。当前 `src/main.tsx` 的 1447-1545 行包含完整的热词管理卡片，包括：

1. 热词列表展示（原词 → 修正词、分类标签、启用状态）
2. 新增热词按钮和对话框表单
3. 编辑热词功能（点击编辑图标打开对话框）
4. 启用/停用热词开关（Switch 组件）
5. 删除热词功能（点击删除图标）
6. 热词上下文预览（显示启用热词的文本整理上下文）

热词管理对话框（1548-1638 行）包含：
- 原始说法输入框
- 修正写法输入框
- 分类输入框
- 启用热词开关
- 保存和取消按钮

后端命令已在 T005 实现：
- `create_hotword`
- `list_hotwords`
- `update_hotword`
- `delete_hotword`
- `enabled_hotword_context`

因此本任务无需新增代码，仅需验证现有功能完整性。

## 为什么这么做

T005 任务在实现热词词典数据层时，同步实现了完整的前端管理界面。这符合项目的开发方式：在实现数据层的同时完成对应的 UI 界面，避免前后端分离导致的集成问题。

热词管理作为卡片展示在主界面中，而不是独立页面，原因：
1. 热词管理是配置类功能，使用频率相对较低
2. 在主界面中展示可以让用户在一个页面内完成所有配置
3. 减少页面切换，提升配置效率
4. 符合当前单页应用的设计模式

### 功能验证清单

需要手动验证以下功能（待 `pnpm tauri dev` 启动后执行）：

1. **新增热词**
   - 点击"新增热词"按钮
   - 填写原始说法、修正写法、分类
   - 保存后检查热词列表是否显示新热词
   - 检查热词上下文预览是否更新

2. **编辑热词**
   - 点击热词项的编辑图标
   - 修改热词内容
   - 保存后检查热词列表是否更新

3. **启用/停用热词**
   - 切换热词的 Switch 开关
   - 检查启用状态是否更新
   - 检查热词上下文预览是否相应变化

4. **删除热词**
   - 点击热词项的删除图标
   - 检查热词是否从列表中移除
   - 检查热词上下文预览是否更新

5. **表单验证**
   - 尝试保存空的原始说法或修正写法
   - 检查是否显示错误提示

6. **热词上下文**
   - 启用多个热词
   - 检查热词上下文预览区域是否正确显示所有启用热词的修正写法
