# XiLuoLin 使用与验证指南

> 本文档面向用户、贡献者和维护者，用于完成本地配置、核心流程验证和问题定位。它不是比赛演示脚本。

## 验证目标

确认 XiLuoLin 能够完成“语音输入 → ASR 转写 → 按人格整理 → 输出文本 → 保存历史 → 更新统计”的桌面工作流，并明确外部服务、系统权限和当前实现边界。

## 前置准备

- Node.js 20+
- pnpm 10+
- Rust stable 工具链
- Windows：Microsoft Visual Studio C++ Build Tools、WebView2 Runtime
- macOS：macOS 13+；当前本地安装包仅支持 Apple Silicon
- 可用的麦克风及系统麦克风权限
- 智谱 GLM-ASR-2512 API Key
- OpenAI Responses API 或兼容文本处理服务的 API Key

## 开发日志中的识别结果

默认日志只记录 ASR / 文本整理所用服务商、模型、结果字符数和是否发生降级，不输出用户口述正文。需要排查识别或整理内容时，可在开发环境显式开启全文日志：

```bash
XILUOLIN_LOG_TEXT=1 pnpm tauri dev
```

全文日志可能包含口述内容、姓名或业务信息，只应在本机临时使用，提交日志前必须脱敏。

## 启动项目

```bash
pnpm install --frozen-lockfile
pnpm check
pnpm tauri dev
```

如果只修改前端，可先运行 `pnpm build`；涉及 Rust、录音、快捷键、凭据或输出能力时，应运行完整的 `pnpm check`，并执行桌面端手动验证。

## 首次配置

1. 打开“设置”页，在“语音输入就绪检查”中确认权限；macOS 用户可在此请求麦克风和辅助功能权限。
2. 配置 ASR 和文本处理 Provider 的 API Key、Base URL 与模型名。
3. 选择麦克风设备。
4. 配置长按模式和切换模式快捷键。
5. 选择输出方式，并确认是否自动保存历史。
6. 在“人格”页选择默认人格，或创建自定义人格。
7. 在“热词”页添加项目名、人名、技术词等易误识别内容。

### Provider 配置说明

应用分别配置语音识别和文本整理服务：

- ASR 当前支持智谱 GLM-ASR-2512 和 OpenAI Whisper 兼容配置。
- 使用智谱 `audio/transcriptions` 时，单次录音应控制在 30 秒以内；更长录音会由服务端拒绝。
- 文本整理支持智谱和 OpenAI Responses API 兼容配置。
- 每类服务都需要选择 Provider，并填写对应的 API Key、Base URL 和模型名。
- 优先通过设置页保存配置，不要直接编辑应用数据文件或把密钥写入仓库。
- 保存后先用短语音验证 ASR，再验证文本整理和跨应用输出，便于定位失败环节。

API Key 通过操作系统凭据库保存；普通配置保存在本地应用数据中。音频和文本会发送到用户主动配置的 Provider。提交 Issue 或日志时，请勿附带密钥、完整录音路径或私人文本。更多恢复步骤见 [`troubleshooting.md`](./troubleshooting.md)。

## 核心流程验证

### 1. 人格与热词

- 确认内置人格可以切换并设为默认。
- 创建一个自定义人格，保存后重新启动应用，确认配置仍然存在。
- 添加并启用一个热词，确认热词上下文参与后续文本整理。

### 2. 语音输入

1. 在任意文本编辑器或聊天输入框中放置光标。
2. 使用设置页显示的全局快捷键开始录音。
3. 口述一段 5 至 15 秒的内容，建议包含已配置热词。
4. 停止录音并等待 ASR 与文本整理完成。
5. 确认结果按照当前人格完成断句、整理和格式化。

推荐测试文本：

```text
帮我把今天关于 XiLuoLin 快捷键可靠性和本地隐私保护的讨论整理成三项开发任务，并为每项补充验收标准。
```

#### 录音状态条

通过全局快捷键录音时，屏幕顶部居中位置会显示鼠标穿透的纯黑胶囊状态条，不会抢走当前输入应用的焦点。胶囊外围保持完全透明。状态条包含录音时长，并依次展示以下阶段：

- 正在录音：时长持续增长，红色状态点和音频条循环运动。
- 正在识别、正在整理、正在输入：录音时长冻结，蓝色音频条提示当前仍在处理。
- 输入完成：显示绿色完成状态，短暂停留后自动消失。
- 处理失败：显示红色失败状态，短暂停留后自动消失；详细错误仍以主窗口提示为准。

状态条在 Windows 和 macOS 上使用透明窗口；其音频条是处理状态动画，不代表麦克风的实时音量。macOS 透明窗口依赖 Tauri 私有 API，因此当前安装包不以 Mac App Store 分发为目标。

### 3. 输出、历史与统计

- 确认结果能够恢复录音开始时的目标应用窗口并写入；精确窗口不可用时可降级为原应用，失败时应降级到剪贴板。
- 确认历史记录包含原始文本、整理结果、人格和录音时长。
- 确认统计卡片在成功处理后更新。
- 重启应用，确认本地历史和设置仍可读取。

## 错误场景

至少验证以下场景并确认界面给出可理解、可恢复的提示：

- 未配置或配置了无效 API Key。
- 麦克风权限被拒绝、尚未请求或设备不可用。
- macOS 辅助功能权限被拒绝，此时自动粘贴应降级到剪贴板。
- Provider 超时、限流或返回非预期响应。
- 全局快捷键冲突。
- 目标应用不支持键盘注入，需要降级到剪贴板。

## 当前实现边界

- 项目当前聚焦短语音到可用文本的桌面输入工作流，不是系统输入法内核。
- 首页可见录音 / 上传入口仍需结合当前 UI 状态确认；全局快捷键是主要输入入口。
- 真实 Provider、麦克风和跨应用输出必须在目标操作系统上手动验证；macOS 构建与安装见 [`macos-build.md`](./macos-build.md)。
- 长音频转写、会议纪要和多人协作不是当前核心范围，但可通过 Issue 提交具体使用场景和设计建议。

## 反馈问题

提交 Issue 前请提供：

- 操作系统、版本与 CPU 架构
- XiLuoLin 版本或 commit
- 可复现步骤、预期行为和实际行为
- 已运行的检查命令
- 已脱敏的错误信息或日志

安全问题不要公开提交，请按照 [SECURITY.md](../SECURITY.md) 的方式报告。
