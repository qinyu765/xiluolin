### 新增内容

在 `src/main.tsx` 的热词卡片后新增"应用设置"卡片，包含：

1. **全局快捷键配置**
   - 输入框用于配置快捷键字符串
   - 提示修饰键格式：CommandOrControl、Alt、Shift
   - 示例：CommandOrControl+Shift+V

2. **录音模式配置**
   - 下拉选择器：长按录音 / 切换式录音
   - 长按：按住快捷键录音，松开停止
   - 切换：按一次开始，再按一次停止

3. **输出方式配置**
   - 下拉选择器：复制到剪贴板 / 自动粘贴
   - 复制：结果复制到剪贴板
   - 自动粘贴：尝试模拟粘贴到当前输入位置

4. **自动保存历史配置**
   - Switch 开关
   - 控制每次语音输入完成后是否自动保存到历史记录

5. **保存按钮**
   - 提交表单保存所有应用设置
   - 调用 `update_app_config` 命令更新配置

### 已有内容

智谱 ASR 和 OpenAI 配置已在 T006 和 T007 中实现，分别作为独立卡片展示：
- T006：智谱 ASR 配置（API Key、Base URL、模型名）

### 配置存储

所有配置通过 `AppConfig` 结构体统一管理，保存到 Tauri Store（`settings.json`）：
- `shortcut`：全局快捷键
- `recording_mode`：录音模式（hold / toggle）
- `output_mode`：输出方式（copy / paste）
- `auto_save_history`：是否自动保存历史

后端命令已在 T003 实现：
- `update_app_config`：更新应用配置
- `read_app_config`：读取应用配置

## 为什么这么做

将应用设置独立为一个卡片，而不是与 ASR 或 OpenAI 配置混在一起，原因：
1. 应用设置（快捷键、录音模式、输出方式）是全局行为配置，与具体服务无关
2. ASR 和 OpenAI 配置是服务端点配置，属于不同的关注点
3. 独立卡片让用户更容易找到和修改应用行为设置
4. 符合"关注点分离"的设计原则

配置保存采用表单提交方式，而不是实时保存，原因：
1. 用户可能需要同时修改多个配置项
2. 避免频繁调用后端命令
3. 给用户明确的保存反馈

### 经验

1. 配置界面应按关注点分离：服务配置（ASR、OpenAI）和应用行为配置（快捷键、录音模式）分开展示
2. 表单提交方式比实时保存更适合多配置项的场景
3. 配置界面应提供清晰的说明文本，帮助用户理解每个配置项的作用
