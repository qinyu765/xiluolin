# T025 人格页整合

## 任务目标

整合"人格管理"和"默认整理风格"两个功能,简化人格字段,优化编辑表单。

## 前置依赖

- T023: 导航框架已建立
- T024: 首页重构完成

## 实际改动

### 前端改动 (src/main.tsx)

1. **移除重复卡片**:
   - 删除"默认整理风格"卡片(T004A UI 基础)
   - 保留"人格管理"卡片,移到"人格"页
   - 移除卡片标题中的"T017 人格页"技术标记

2. **简化人格展示**:
   - 内置人格区:只显示名称、描述
   - 自定义人格区:只显示名称、描述
   - 移除详细字段展示(适用场景、输出语气、输出结构)
   - 这些字段在编辑时仍然保留,但列表中不展示

3. **优化人格编辑表单**:
   - 必填字段:名称、描述、提示词
   - 可选字段:适用场景、输出语气、输出结构
   - 添加字段说明:
     - 名称:简短的人格名称,如"技术文档助手"
     - 描述:一句话说明这个人格的用途
     - 提示词:指导 AI 整理文本的系统提示词
     - 适用场景、输出语气、输出结构:可选,用于补充说明

4. **优化默认人格选择**:
   - 在人格列表中,默认人格显示"默认"标签
   - 非默认人格显示"设为默认"按钮
   - 点击后立即生效,无需额外保存

## 为什么这么做

### 架构选择

1. **合并重复功能**:
   - "人格管理"和"默认整理风格"本质上是同一功能
   - 合并后减少用户认知负担
   - 统一的人格管理入口

2. **简化展示字段**:
   - 列表中只显示核心信息:名称、描述
   - 详细字段在编辑时可见
   - 减少视觉噪音,提升可读性

3. **保留可选字段**:
   - 适用场景、输出语气、输出结构对高级用户有用
   - 但不是必填,降低新建人格的门槛
   - 这些字段可以合并到描述中

### UI 设计

1. **人格列表布局**:
   - 内置人格和自定义人格分开展示
   - 每个人格卡片:名称、描述、默认标签、操作按钮
   - 内置人格不可编辑或删除,只能设为默认

2. **人格编辑表单**:
   - 必填字段在前,可选字段在后
   - 每个字段添加占位符和说明文字
   - 提示词使用 Textarea,支持多行输入

3. **默认人格标识**:
   - 使用小标签显示"默认"
   - 视觉上与其他人格区分
   - 点击"设为默认"后,标签自动切换

1. **字段说明一致性**：后续可以考虑在所有表单中统一使用书名号或其他符号来标记示例文本。
