# XiLuoLin 演示脚本

> 本文档用于评审演示和录屏准备，基于 2026-05-25 的 `dev` 分支状态编写。

## 演示目标

用 3 到 5 分钟说明 XiLuoLin 的核心价值：把自然口述转换成可直接使用的办公、写作和编程文本，并通过人格、热词、历史和统计降低后续编辑成本。

## 演示前准备

- 已安装 Node.js 20+、pnpm 10+、Rust 工具链、Microsoft Visual Studio C++ Build Tools 和 WebView2 Runtime。
- 已执行 `pnpm install`。
- 已准备智谱 GLM-ASR-2512 API Key 和 OpenAI Responses API Key。
- 已准备一段 5 到 15 秒的中文短音频，内容建议包含项目名或技术词，例如“把七牛云黑客松语音输入助手的演示流程整理成三步”。
- 麦克风权限已开启，系统输入法和剪贴板可正常使用。

## 演示流程

### 1. 项目定位

打开应用后，先说明：

- XiLuoLin 是桌面语音输入助手，MVP 聚焦短语音输入，不做完整系统级输入法内核。
- 核心闭环是“语音输入 → ASR 转写 → 人格化整理 → 复制或自动粘贴 → 历史保存 → 统计更新”。
- 目标用户是开发者、团队协作者和创作者。

### 2. 设置服务

进入“设置”页：

- 在“通用”中展示长按模式快捷键、切换模式快捷键、麦克风设备、输出方式和自动保存历史。
- 在“模型配置”中展示智谱 GLM-ASR-2512 和 OpenAI Responses API 配置。
- 强调 API Key 本地保存，不提交到仓库；音频和文本只发送到用户配置的服务。

### 3. 配置人格

进入“人格”页：

- 展示 4 个内置人格：Prompt 工程师、任务协作者、灵感整理师、正式消息助手。
- 新建一个自定义人格，例如“黑客松答辩助手”，提示词要求输出短标题、三条亮点和一句收束语。
- 将自定义人格设为默认。

### 4. 配置热词

进入“热词”页：

- 新增热词，例如“西洛林 → XiLuoLin”“七牛 → 七牛云”“GLM ASR → GLM-ASR-2512”。
- 启用热词后，展示热词上下文如何进入文本整理 prompt。
- 说明热词用于减少专有名词、项目名和技术词误识别。

### 5. 语音输入闭环

回到“首页”：

- 展示当前人格、快捷键提示、统计卡片和时间分段历史记录。
- 执行一次短语音输入，等待 ASR 和 OpenAI 整理完成。
- 展示原始识别文本和整理结果。
- 点击复制或输出，说明自动粘贴失败时会回退到剪贴板。
- 查看历史记录新增项和统计卡片更新。

## 推荐口述样例

```text
帮我把七牛云黑客松这个语音输入助手的演示流程整理成一段清晰的介绍，重点说明它能把口头想法转成可以直接发给 Agent 的 Prompt，还能用热词保证项目名和技术词不被识别错。
```

## 评审讲解要点

- 产品完整性：已覆盖配置、人格、热词、语音处理、输出、历史和统计的 MVP 闭环。
- 创新点：把普通语音输入升级为“按场景整理后的可用文本”，重点服务 Agent Prompt、任务协作和创作记录。
- 工程质量：Tauri + React 架构清晰，业务数据本地 SQLite 保存，服务调用通过 Provider 模块隔离。
- 隐私边界：API Key、历史、人格和热词都保存在本地；仓库不包含真实密钥或录音文件。
