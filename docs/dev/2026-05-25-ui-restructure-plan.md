# UI 重构计划

> **归档说明：** 本文记录特定开发阶段的背景与决策，其中的 MVP、demo、比赛或旧分支流程表述仅用于保留历史，不代表 XiLuoLin 当前的开源项目定位与协作方式。当前信息请以根目录 `README.md`、`CONTRIBUTING.md` 和 `docs/roadmap.md` 为准。

> 当前文档是 UI 重构启动时的计划稿。实际实现已经发生偏差：全局导航采用左侧导航；设置页采用顶部水平 Tab；首页 `QuickStartCard` 录音 / 上传入口当前被注释隐藏。当前事实源以 `README.md`、`docs/solution-design.md` 第 9 节和 `docs/dev/2026-05-25-t028-ui-e2e-verification.md` 为准。

## 问题背景

当前主界面 (src/main.tsx) 将所有功能卡片垂直堆叠在一个页面中,存在以下问题:

1. **缺少导航结构**:没有左侧 Tab 栏,所有内容混在一起,用户难以快速定位功能
2. **信息冗余**:存在大量无用文本提示,如"面向办公、写作和编程场景...","T015 主界面"等技术标记
3. **功能重复**:人格管理页和默认整理风格页功能重叠,需要整合
4. **热词定义不清**:热词功能的实际意义和作用需要重新定义
5. **设置页布局混乱**:设置项过多,缺少分类和层级结构
6. **历史记录功能不完整**:只支持复制,缺少删除功能;没有时间分段展示

### 页面结构

#### 1. 首页 (Home)

包含两个主要区域:

**快速开始区**:
- 当前人格选择器
- 录音按钮(大型圆形按钮)
- 录音状态和时长显示
- 原始识别文本展示
- 整理结果展示
- 复制和输出按钮
- 音频文件上传(作为备选方案)

当前状态：上述快速开始区已抽取为 `QuickStartCard`，但当前在 `HomePage` 中被注释隐藏。

**语音输入成效区**:
- 统计卡片:协作次数、累计时间、生成字数、节省时间、常用人格
- 历史记录列表:
  - 按时间分段显示(今天、昨天、具体日期)
  - 每条记录显示:人格名称、时间、时长、字数
  - 操作按钮:复制、删除
  - 点击展开查看完整文本

#### 2. 人格页 (Persona)

整合原"人格管理"和"默认整理风格"功能:

**内置人格区**:
- 展示 4 个内置人格
- 显示:名称、描述
- 标记默认人格
- 支持设为默认

**自定义人格区**:
- 展示用户创建的人格
- 显示:名称、描述
- 操作:编辑、删除、设为默认
- 新建人格按钮

**人格编辑表单**(简化):
- 必填:名称、描述、提示词
- 可选:适用场景、输出语气、输出结构(可以合并到描述中)

#### 3. 热词页 (Hotword)

重新定义热词功能:"记住那些只属于你的重要词汇"

**功能定位**:
- 专有名词修正:项目名、产品名、人名、地名
- 技术词汇:框架名、库名、API 名称
- 行业术语:特定领域的专业词汇
- 个人习惯:常用缩写、昵称

**界面设计**:
- 热词列表:原始说法 → 修正写法
- 分类标签:技术词、项目名、人名等
- 操作:新增、编辑、删除、启用/停用
- 搜索和筛选功能(可选)

**实现方式**:
- 启用的热词作为上下文注入到文本整理 prompt
- 格式:"用户常用词汇: Next.js(框架), XiLuoLin(项目名)..."

#### 4. 设置页 (Settings)

计划阶段设想为左侧 Tab 栏分类，当前实际实现为顶部水平 Tab:

**通用设置**:
- 快捷键配置:长按模式、切换模式
- 录音模式:长按录音 / 切换式录音
- 麦克风设备选择
- 录音时静音其他应用
- 输出方式:复制到剪贴板 / 自动粘贴
- 自动保存历史

**模型配置**:
- 智谱 GLM-ASR-2512:API Key、Base URL、模型名
- OpenAI Responses API:API Key、Base URL、模型名

### 历史记录时间分段

```typescript
function groupHistoryByDate(records: HistoryRecord[]) {
  const today = new Date();
  const yesterday = new Date(today);
  yesterday.setDate(yesterday.getDate() - 1);
  
  return {
    today: records.filter(r => isSameDay(r.created_at, today)),
    yesterday: records.filter(r => isSameDay(r.created_at, yesterday)),
    older: records.filter(r => !isSameDay(r.created_at, today) && !isSameDay(r.created_at, yesterday))
  };
}
```

## 设计原则

1. **简洁优先**:移除所有技术标记和冗余文本
2. **功能聚合**:相关功能放在同一页面
3. **层级清晰**:主导航 → 次级 Tab → 内容区
4. **操作直观**:常用操作一键可达
5. **信息密度适中**:避免单页信息过载
